# Nginx Docker Projects

Two ready-to-use Nginx configurations: a simple HTTP server and a full SSL/TLS setup for development and production.

## 📁 Project Structure

```
nginx-docker/
├── docker-compose.yml          # Basic HTTP Nginx (port 8080)
├── Dockerfile
├── nginx.conf
└── index.html

nginx-ssl-docker/
├── docker-compose.yml          # Development (self-signed SSL)
├── docker-compose.prod.yml     # Production (Let's Encrypt)
├── Dockerfile
├── Dockerfile.prod
├── nginx/
│   ├── nginx.conf              # Main config (gzip, performance, logging)
│   ├── ssl_self.conf           # Dev: self-signed certificate config
│   └── ssl.conf                # Prod: Let's Encrypt config
├── ssl/self-signed/            # Dev certificates (generated by setup-dev.sh)
├── certbot/                    # Let's Encrypt certificates
├── html/                       # Static website content
├── setup-dev.sh                # Generate self-signed certs, build dev image
└── setup-prod.sh               # Production setup helper
```

## 🚀 Quick Start

### Option 1: Basic Nginx (HTTP only)
```bash
cd nginx-docker
docker compose up -d
# Access: http://localhost:8080
```

### Option 2: Nginx with SSL (development)
```bash
cd nginx-ssl-docker
chmod +x setup-dev.sh
./setup-dev.sh
docker compose up -d
# HTTP:  http://localhost  (redirects to HTTPS)
# HTTPS: https://localhost
```

### Option 3: Nginx with SSL (production / Let's Encrypt)
```bash
cd nginx-ssl-docker
chmod +x setup-prod.sh init-letsencrypt.sh

# Edit nginx/ssl.conf — replace yourdomain.com with your domain
# Edit init-letsencrypt.sh — set your email address

./init-letsencrypt.sh yourdomain.com
docker compose -f docker-compose.prod.yml up -d
```

## 🔧 Configuration Details

### Port Mappings

| Project | Environment | HTTP | HTTPS |
|---------|-------------|------|-------|
| `nginx-docker` | Basic | 8080 | — |
| `nginx-ssl-docker` | Development | 80 → 443 | 443 |
| `nginx-ssl-docker` | Production | 80 → 443 | 443 |

### SSL Certificates

**Development (self-signed)**
- Location: `ssl/self-signed/cert.pem` and `key.pem`
- Generated by: `setup-dev.sh`
- Validity: 365 days, CN: localhost
- Browser will show a security warning — this is expected in dev

**Production (Let's Encrypt)**
- Location: `certbot/conf/`
- Renewed automatically every 12 hours
- Validity: 90 days (auto-renewed well before expiry)

### Security Headers (ssl_self.conf)

The dev SSL config already includes:
```nginx
add_header X-Frame-Options "SAMEORIGIN" always;
add_header X-Content-Type-Options "nosniff" always;
add_header X-XSS-Protection "1; mode=block" always;
```

For production, also add to `ssl.conf`:
```nginx
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
add_header Referrer-Policy "no-referrer-when-downgrade" always;
add_header Content-Security-Policy "default-src 'self' https: data: 'unsafe-inline'" always;
```

### Health Check Endpoint

Both SSL configs expose `/health` for container health checks:
```bash
curl http://localhost/health        # returns "healthy"
curl https://localhost/health -k    # dev (skip cert warning)
```

This is wired up as the `healthcheck` in both compose files.

## 🛠️ Common Commands

```bash
# Start / stop
docker compose up -d
docker compose down

# View logs
docker compose logs -f

# Rebuild image after config changes
docker compose build --no-cache && docker compose up -d

# Validate Nginx config without restarting
docker compose exec nginx-dev nginx -t

# Reload config (no downtime)
docker compose exec nginx-dev nginx -s reload
```

## 🔐 SSL Certificate Commands

```bash
# Check certificate expiry date
docker compose exec nginx-dev \
  openssl x509 -in /etc/nginx/ssl/self-signed/cert.pem -noout -dates

# Test SSL from outside the container
openssl s_client -connect localhost:443 -servername localhost

# Force Let's Encrypt renewal (production)
docker compose -f docker-compose.prod.yml run --rm certbot renew --force-renewal
```

## 🐛 Troubleshooting

### Certificates Not Found
```bash
ls -la nginx-ssl-docker/ssl/self-signed/

# If empty, regenerate manually:
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout ssl/self-signed/key.pem \
  -out ssl/self-signed/cert.pem \
  -subj "/C=US/ST=Dev/L=Local/O=Dev/CN=localhost"

chmod 644 ssl/self-signed/cert.pem
chmod 600 ssl/self-signed/key.pem
```

### Port Conflicts
```bash
sudo lsof -i :80
sudo lsof -i :443
sudo lsof -i :8080
# Change the port mapping in docker-compose.yml if needed
```

### Container Won't Start
```bash
docker compose logs nginx-dev
docker compose exec nginx-dev nginx -t     # check config syntax
ls -la nginx/                              # check file permissions
```

### HTTPS Not Redirecting
Ensure the HTTP server block in `ssl_self.conf` has the redirect and that port 80 is mapped in `docker-compose.yml`.

## 📈 Performance Tips

1. **Gzip compression** is already enabled in `nginx.conf`
2. **HTTP/2** is enabled via `listen 443 ssl http2`
3. **SSL session caching** (`ssl_session_cache shared:SSL:10m`) reduces handshake overhead
4. For production, add cache-control headers for static assets:
   ```nginx
   location ~* \.(js|css|png|jpg|ico|woff2)$ {
       expires 1y;
       add_header Cache-Control "public, immutable";
   }
   ```
5. Use a CDN for static assets in high-traffic production deployments

## 🔒 Security Best Practices

1. **Never commit private keys or certificates** — `.gitignore` already excludes `*.pem`, `*.key`, `certbot/`
2. **TLS 1.2 minimum** — TLSv1.0 and TLSv1.1 are disabled in both SSL configs
3. **OCSP stapling** — add to production config for faster TLS handshakes:
   ```nginx
   ssl_stapling on;
   ssl_stapling_verify on;
   ```
4. **Rate limiting** — add to production config:
   ```nginx
   limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;
   limit_req zone=api burst=20 nodelay;
   ```
5. **Diffie-Hellman parameters** — generate for production:
   ```bash
   openssl dhparam -out ssl/dhparam.pem 2048
   ```
   Then reference in `ssl.conf`: `ssl_dhparam /etc/nginx/ssl/dhparam.pem;`
